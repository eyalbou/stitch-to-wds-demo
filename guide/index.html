Architecture

How it works

Stitch generates high-fidelity screens. The pipeline converts them to WDS. CK delivers them in the existing skill flow.

1

CK Skill

Gathers spec, strategy, research, and concept context

→
2

Stitch MCP

Generates screens from text prompts via Google Stitch AI

→
3

WDS Convert

Swaps fonts, colors, buttons, icons to WDS tokens

→
4

CK Output

Saved to artifacts/ for edge cases, content, and handoff

What changes

ck-ux-concepts and ck-ux-all-screens gain a third generation mode: Stitch AI. You choose it alongside HTML and Figma.

What doesn't change

Existing HTML and WDS React modes work exactly as before. Stitch is opt-in. Orchestrators are untouched. Artifact paths are preserved.

What CK gives Stitch

Product spec, strategy, research insights, and current-product screenshots all feed into better Stitch prompts -- richer context, better output.

Setup

Prerequisites

One-time setup before using Stitch mode.

✓

Google Cloud authentication

Run gcloud auth login and gcloud auth application-default login in your terminal. Free tier covers Stitch usage (350 generations/month).

✓

Stitch MCP server

Add to ~/.cursor/mcp.json:

"user-stitch": { "command": "npx", "args": ["-y", "@anthropic-ai/stitch-mcp"] }

✓

WDS MCP server

Add to ~/.cursor/mcp.json:

"wix-wds": { "type": "http", "url": "https://mcp.wix.com/mcp" }

✓

Step by step

Usage: ck-ux-concepts

Generating UX concepts with Stitch mode -- 3 decision points.

1

Invoke the skill

Say "generate UX concepts" or "ck-ux-concepts". The skill finds your spec + strategy docs and runs silent analysis as usual.

2

Choose delivery mode

You're asked: "All at once" or "One by one". Same as before -- this controls how concepts are presented.

3

Choose generation mode NEW

A new Step 1C asks you to pick:
HTML/CSS/JS (default) -- agent writes code inline, B&W wireframes
Stitch AI -- generates via Google Stitch MCP, auto-converted to WDS
Figma -- coming soon

4

Choose color mode (Stitch only) NEW

If you picked Stitch, you're asked: B&W wireframes (CK convention) or WDS-colored (full fidelity). If you already said "in color" or "wireframes", this is skipped.

5

Concepts are generated

For each concept, Stitch generates a screen, the WDS conversion pipeline runs, and the result is saved to both artifacts/ux/concepts/ and .stitch/{slug}/. Documentation and delivery follow the same CK flow.

Usage: ck-ux-all-screens

Full UX specs with Stitch mode -- same skill, new option.

1

Select concept + design mode

Step 0.2 now shows three design modes: Keep concept theme, Use WDS components, and Generate with Stitch AI.

2

Edge-case baton

In Stitch mode, the agent builds a prompt baton from the selected concept + edge case analysis. Each remaining screen gets a Stitch prompt that includes the concept's design language and the specific flow's edge case states.

3

Screen-by-screen generation

Each flow is generated via Stitch MCP, WDS-converted, annotated with the drawer system, and wired with cross-file navigation. Output goes to both CK and Stitch paths.

Design choice

B&W wireframes vs WDS-colored

CK concepts are traditionally B&W to keep focus on layout. Stitch adds the option of full-color WDS output.

B&W Wireframes

Grayscale palette. Focuses review on layout, hierarchy, and information architecture. CK convention for early-stage concepts.

CK Default

WDS-Colored

Full Wix Design System brand colors (#116DFF primary). Production-ready fidelity. Skips a step if going straight to handoff.

Full Fidelity
What you get

Output structure

Stitch mode writes to both CK artifact paths and the Stitch pipeline directory.

artifacts/ux/concepts/ ← CK skills read from here
  concepts.html
  concepts.json
  concept-1-dashboard-hub/
    index.html ← WDS-adjusted output
    concept-1-dashboard-hub.md
  concept-2-spotlight-search/
    index.html

.stitch/{slug}/ ← Stitch pipeline reads from here
  metadata.json
  designs/
    concept-1.html ← Raw Stitch output
    concept-1.png
  wds-output/
    concept-1-adjusted.html ← WDS tokens swapped
    concept-1-converted.tsx ← Full WDS React code
    concept-1-report.md
    concepts-comparison.html ← Tabbed viewer

Output A: WDS-Adjusted

Same layout as the Stitch original. Fonts replaced with Wix Madefor Display, colors mapped to WDS tokens, buttons pill-shaped, Material Icons swapped to wix-ui-icons-common. Quick win -- preserves the designer's layout intent.

Output B: WDS Conversion

Full restructure using <Page>, <Card>, <Table>, <Tabs> WDS components. React/TSX code ready for production. SP tokens, dataHooks, and proper WDS composition patterns.

Integration map

Where Stitch fits in Creator Kit

Stitch slots into Phase 4 (Design) as a generation mode. Earlier phases feed it context; later phases consume its output.

Skills that enrich Stitch

These feed valuable context into Stitch prompts:

ck-product-spec + strategy -- structured requirements become prompts
ck-ux-research -- competitive analysis and user insights
ck-ux-current-product -- existing product screenshots as input
ck-ux-edge-cases -- edge case states for each screen
ck-ux-content -- refined copy before generation

Skills Stitch enhances

These benefit from higher-fidelity Stitch output:

ck-ux-concepts -- Stitch produces richer visual prototypes than hand-written HTML
ck-ux-all-screens -- full UX specs with more polished screens
ck-handoff -- gains .tsx files and conversion reports in the handoff doc
