One-time setup

Prerequisites

Before using Stitch mode, complete these one-time steps.

✓

Google Cloud authentication

Run gcloud auth login and gcloud auth application-default login. Free tier covers Stitch (350 generations/month).

✓

Stitch MCP server in Cursor

Add to ~/.cursor/mcp.json: "user-stitch": { "command": "npx", "args": ["-y", "@anthropic-ai/stitch-mcp"] }

✓

WDS MCP server in Cursor

Add to ~/.cursor/mcp.json: "wix-wds": { "type": "http", "url": "https://mcp.wix.com/mcp" }

✓

Creator Kit with Stitch branch

Checkout eyalbou/stitch-integration on wix-private/creator-kit

End-to-end walkthrough

The 6-Phase Pipeline with Stitch

The Creator Kit product development pipeline has 6 phases. Stitch integrates at Phase 4 and Phase 5 -- everything else is unchanged.

Existing CK phase (unchanged)
Stitch integration point (you choose here)
Enriched by Stitch output (auto)
1

Initiation

/ck-new
Say "/ck-new" to start a project. Creates the folder structure and basic scaffolding.
No changes here -- works exactly as before.
2

Discovery

/ck-flow-discovery
Competitive analysis, user pain points, UX patterns from the market. Runs ck-research-* and ck-ux-research skills.
Output: artifacts/ux/research/
Research insights are woven into Stitch prompts for grounded, differentiated designs.
3

Product Sculpting

/ck-flow-product
Defines spec (requirements, goals, user stories) and strategy (positioning, success metrics, target users).
Runs ck-product-spec + ck-product-strategy skills. Optionally builds a B&W replica of the existing product (ck-ux-current-product).
Output: artifacts/product/product-spec-*.md, artifacts/product/product-strategy-*.md
Better spec & strategy = better Stitch output. These docs feed into prompts.
4

UX Concepts

/ck-flow-ux-concepts Stitch integration
This is the main integration point. The flow generates 5 distinct UX concepts. After silent analysis, you're asked:

Decision 1 -- Delivery mode: "All at once" or "One by one"
Decision 2 -- Generation mode:
  • HTML/CSS/JS (default) -- agent writes code inline, B&W wireframes
  • Stitch AI -- generates via Google Stitch MCP, auto-converted to WDS
  • Figma -- coming soon

Decision 3 -- Color mode (Stitch only):
  • B&W wireframes -- CK convention, grayscale, layout-focused review
  • WDS-colored -- full fidelity, production-ready WDS palette

For each concept, Stitch generates a screen, the WDS conversion pipeline runs automatically, and the result is saved to artifacts/ux/concepts/. Docs and delivery follow the same CK flow.

Output: artifacts/ux/concepts/concept-1-*/ through concept-5-*/
5

All Screens

/ck-flow-final-ux Stitch integration
Takes the selected concept and generates ALL flows, states, and screens. Includes edge case analysis (ck-ux-edge-cases) and content refinement (ck-ux-content). You're asked to choose a design mode:

  • Keep concept theme (default) -- HTML inheriting the concept's style
  • Use WDS components -- Vite + React with real @wix/design-system
  • Generate with Stitch AI -- higher fidelity, WDS-converted

In Stitch mode, the agent builds an edge-case baton: each remaining screen gets a Stitch prompt built from the selected concept's design language + the edge case states for that specific flow. Screens are generated one by one via Stitch MCP, WDS-converted, annotated, and wired together.

Output: artifacts/ux/all-screens/*.html + all-screens.html viewer
6

Handoff

/ck-handoff Enriched
Packages everything into a developer handoff doc: spec, strategy, designs, edge cases, checklist.

When Stitch was used, the handoff doc automatically gains a "WDS Components" section with:
  • .tsx component files (production-ready WDS React code)
  • Conversion reports (token mapping, component mapping)
  • Comparison viewer links (original vs WDS-adjusted vs WDS-converted)

Output: artifacts/handoff/handoff-*.md
Phase 4 detail

B&W wireframes vs WDS-colored

When you pick Stitch AI in Phase 4 (UX Concepts), you choose the visual fidelity.

B&W Wireframes

Grayscale. Review focuses on layout and information architecture. CK convention for early-stage concepts.

CK Default

WDS-Colored

Full Wix Design System brand colors. Production-ready fidelity. Skips a step to handoff.

Full Fidelity
What you get

Output structure

Stitch writes to CK artifact paths (so downstream skills work) and the Stitch pipeline directory (for iteration).

artifacts/ux/concepts/ ← CK skills read from here (phases 5-6)
  concepts.html ← viewer with all 5 concepts
  concept-1-dashboard-hub/
    index.html ← WDS-adjusted output
    concept-1-dashboard-hub.md

.stitch/{slug}/wds-output/ ← Stitch pipeline (for iteration)
  concept-1-adjusted.html ← same layout, WDS tokens
  concept-1-converted.tsx ← full WDS React code
  concept-1-report.md ← conversion report
  concepts-comparison.html ← tabbed side-by-side viewer

Output A: WDS-Adjusted

Same layout as Stitch original. Fonts swapped to Wix Madefor Display, colors to WDS tokens, buttons pill-shaped, icons from wix-ui-icons-common. Preserves the designer's layout.

Output B: WDS Conversion

Full restructure using <Page>, <Card>, <Table>, <Tabs> WDS components. React/TSX production code with SP tokens and dataHooks.
