One-time setup

Prerequisites

Before using Stitch mode, complete these one-time steps.

✓

Google Cloud authentication

Run gcloud auth login and gcloud auth application-default login. Free tier covers Stitch (350 generations/month).

✓

Stitch MCP server in Cursor

Add to ~/.cursor/mcp.json: "user-stitch": { "command": "npx", "args": ["-y", "@anthropic-ai/stitch-mcp"] }

✓

WDS MCP server in Cursor

Add to ~/.cursor/mcp.json: "wix-wds": { "type": "http", "url": "https://mcp.wix.com/mcp" }

✓

CK Stitch extensions in your project

Copy 2 extension files into your project: .agents/skills/ext-ck-flow-ux-concepts/SKILL.md and .agents/skills/ext-ck-ux-concepts/SKILL.md. No core CK changes needed.

✓

WDS pipeline files

Place wds-convert.js and wds-design-system-template.json in .stitch/ at your project root.

End-to-end walkthrough

The 6-Phase Pipeline with Stitch

The Creator Kit product development pipeline has 6 phases. Stitch integrates at Phase 4 and Phase 5 -- everything else is unchanged.

Existing CK phase (unchanged)
Stitch integration point (you choose here)
Enriched by Stitch output (auto)
1

Initiation

/ck-new
Say "/ck-new" to start a project. Creates the folder structure and basic scaffolding.
No changes here -- works exactly as before.
2

Discovery

/ck-flow-discovery
Competitive analysis, user pain points, UX patterns from the market. Runs ck-research-* and ck-ux-research skills.
Output: artifacts/ux/research/
Research insights are woven into Stitch prompts for grounded, differentiated designs.
3

Product Sculpting

/ck-flow-product
Defines spec (requirements, goals, user stories) and strategy (positioning, success metrics, target users).
Runs ck-product-spec + ck-product-strategy skills. Optionally builds a B&W replica of the existing product (ck-ux-current-product).
Output: artifacts/product/product-spec-*.md, artifacts/product/product-strategy-*.md
Better spec & strategy = better Stitch output. These docs feed into prompts.
4

UX Concepts

/ck-flow-ux-concepts Stitch integration
This is the main integration point. The flow generates 3-5 UX concepts. When the CK extensions are installed, a new question appears:

"How should UX concepts be generated?"
  • A: B&W prototypes (default CK) -- agent designs + codes HTML/CSS/JS, strict #000/#fff wireframes
  • B: Stitch Creative -- Stitch AI designs AND renders from loose prompts, full WDS color. Stitch proposes its own layouts.
  • C: Stitch Directed -- agent designs the concepts (same ideas as B&W), Stitch renders them with full WDS color.

What the comparison shows:
  • A vs C -- same concept ideas, different visual quality → Stitch as a renderer
  • A vs B -- different ideas AND quality → Stitch as a designer
  • B vs C -- both WDS, different creative direction → agent ideas vs Stitch ideas

Output: artifacts/ux/demo/stage4/{bw,stitch-creative,stitch-directed}/
5

All Screens

/ck-flow-final-ux Stitch integration
Takes the winning concept from each Stage 4 branch and generates additional screens: search results, account detail, empty state, confirmation modal.

Each branch continues independently:
  • A → B&W wireframe screens (agent codes #000/#fff)
  • B → Stitch Creative screens (Stitch's design language, WDS color)
  • C → Stitch Directed screens (agent's concepts, Stitch quality, WDS color)

All Stitch-generated screens go through the same wds-convert.js pipeline for WDS compliance.

Output: artifacts/ux/demo/stage5/{bw,stitch-creative,stitch-directed}/
6

Handoff

/ck-handoff Enriched
Packages everything into a developer handoff doc: spec, strategy, designs, edge cases, checklist.

When Stitch was used, the handoff doc is enriched with:
  • WDS-adjusted HTML files (fonts, colors, buttons, layout all converted to WDS tokens)
  • Comparison viewer links (original vs WDS-adjusted concepts)
  • WDS component mapping notes (HTML elements mapped to @wix/design-system React components for production)

Output: artifacts/handoff/handoff-*.md
Phase 4 detail

Three Generation Modes

Each mode answers a different question: Is Stitch a better renderer? A better designer? Or both?

A: B&W (Original CK)

Agent designs + codes. Strict #000/#fff wireframes with working JS. Focus on layout and IA.

CK Default

B: Stitch Creative

Stitch AI designs AND renders. Loose prompts, Stitch proposes its own layouts. Full WDS color.

Stitch as Designer

C: Stitch Directed

Agent designs, Stitch renders. Same concepts as B&W but in full WDS color. Best of both worlds.

Stitch as Renderer
What you get

Output structure

Stitch writes to CK artifact paths (so downstream skills work) and the Stitch pipeline directory (for iteration).

artifacts/ux/demo/ ← Three-branch demo output
  stage4/
    bw/ ← Branch A: 3 B&W wireframe concepts
    stitch-creative/ ← Branch B: 3 Stitch-designed concepts
    stitch-directed/ ← Branch C: 3 agent concepts, Stitch rendered
  stage5/
    bw/ ← Branch A: 4 screens (B&W)
    stitch-creative/ ← Branch B: 4 screens (Stitch design)
    stitch-directed/ ← Branch C: 4 screens (Stitch rendered)
  stage4-bw.html ← viewer: B&W concepts
  stage4-stitch-creative.html ← viewer: Stitch Creative concepts
  stage4-stitch-directed.html ← viewer: Stitch Directed concepts
  stage5-bw.html ← viewer: B&W screens
  stage5-stitch-creative.html ← viewer: Stitch Creative screens
  stage5-stitch-directed.html ← viewer: Stitch Directed screens

WDS-Adjusted HTML

Same layout as Stitch original. 7-pass pipeline applies: font swap to Wix Madefor Display, MD3 color tokens mapped to WDS hex values, buttons styled to WDS spec (rounded-lg, correct skins), input/select normalization, Tailwind shadow cleanup, and typography class alignment.

Comparison Viewer

Tabbed HTML page showing original UI screenshot alongside all generated concepts. Each tab renders the full-page HTML at consistent size for easy comparison. Navigate concepts with a single click.

Stage 5: Continuing Each Branch

Same three-tab setup for the expanded screens (search results, account detail, empty state, confirmation modal):

Tab 2 -- B: Stitch Creative screens: 4 screens following Stitch's own design language from Stage 4.

Tab 3 -- C: Stitch Directed screens: 4 screens matching the agent's concept layout, rendered by Stitch with WDS color.

What the three-way comparison shows

Each pair reveals a different Stitch capability:

• A vs C (same ideas, different quality): Stitch as a renderer -- takes your designs and makes them production-ready
• A vs B (different ideas, different quality): Stitch as a designer -- brings its own creative direction
• B vs C (both WDS, different direction): whose ideas win? Agent's structured approach vs Stitch's AI intuition
